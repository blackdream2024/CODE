## 1. 项目概述

### 1.1 项目简介

「命理星河」是一个基于中国传统命理学的八字命盘分析系统，融合了 OASIS 多智能体社会仿真系统和 N-S-M 概率云模型，为用户提供专业的命理分析服务。

**Slogan**：星辰演化，命运可期

### 1.2 核心功能

| 功能模块 | 说明 |
|---------|------|
| 命盘录入 | 录入出生信息，生成八字命盘 |
| 命盘查看 | 查看八字排盘、五行分析、十神分析 |
| 推演仿真 | 基于 OASIS 多智能体社会仿真，输出概率云 |
| 关系分析 | 分析两人八字合婚、关系走势 |
| 风水分析 | 分析住宅风水格局、吉凶方位 |

### 1.3 设计风格

- **主色调**：活力橙 (#d4a853) + 深炭黑 (#0a0a1a)
- **设计语言**：极简主义，中国传统命理风格
- **交互设计**：直观的表单输入，清晰的结果展示

### 4.2 命盘录入页面 (chart-input)

**功能描述**：录入出生信息，生成八字命盘。

**主要功能**：
- 姓名输入
- 性别选择（男/女）
- 出生日期选择
- 出生时辰选择（13个时辰）
- 出生地输入
- 历法选择（公历/农历）
- 命盘保存到本地存储
- 已录入命盘列表展示

**表单字段**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| name | 文本 | 是 | 姓名 |
| gender | 选择 | 是 | 性别（男/女） |
| birthDate | 日期 | 是 | 出生日期 |
| birthTime | 时间 | 是 | 出生时辰 |
| birthPlace | 文本 | 否 | 出生地 |
| calendar | 选择 | 否 | 历法（公历/农历） |

**数据存储**：
```javascript
// 命盘数据结构
{
  id: '1234567890',           // 唯一标识
  name: '张三',               // 姓名
  gender: 'male',             // 性别
  birth_date: '1990-01-01',   // 出生日期
  birth_time: '子时',         // 出生时辰
  birth_place: '北京',        // 出生地
  calendar: '公历',           // 历法
  created_at: '2026-05-25T10:00:00.000Z'  // 创建时间
}
```

**本地存储**：
- 使用 `wx.getStorageSync('charts')` 获取命盘列表
- 使用 `wx.setStorageSync('charts', charts)` 保存命盘列表

### 4.4 推演仿真页面 (simulation)

**功能描述**：基于 OASIS 多智能体社会仿真，输出概率云。

**主要功能**：
- 命盘选择
- 推演场景选择（7种场景）
- 起始时间配置
- 仿真步数配置
- 采样次数配置
- 推演结果展示

**推演场景**：

| 场景 | 英文标识 | 说明 |
|------|----------|------|
| 事业推演 | career | 分析事业发展趋势 |
| 婚姻推演 | marriage | 分析婚姻感情走势 |
| 合作推演 | cooperation | 分析合作关系 |
| 搬迁推演 | relocation | 分析搬迁运势 |
| 投资推演 | investment | 分析投资风险 |
| 健康推演 | health | 分析健康状况 |
| 学习推演 | learning | 分析学习运势 |

**API 请求参数**：
```javascript
{
  agents: [{
    name: '张三',
    birth_date: '1990-01-01',
    birth_time: '子时',
    gender: 'male'
  }],
  scenario: 'career',      // 推演场景
  steps: 12,               // 仿真步数（月）
  samples: 50,             // 采样次数
  start_year: 2026,        // 起始年份
  start_month: 5           // 起始月份
}
```

**结果展示**：
- 统计卡片（仿真月数、关键决策点、风险等级）
- 风险分析（风险等级、风险指数）
- 推演总结
- 关键决策点（高峰/低谷）
- 专业建议

## 7. 开发规范

### 7.1 代码规范

#### 命名规范
- **页面文件**：使用 kebab-case（如：`chart-input`）
- **变量命名**：使用 camelCase（如：`birthDate`）
- **常量命名**：使用 UPPER_SNAKE_CASE（如：`API_BASE_URL`）
- **CSS 类名**：使用 kebab-case（如：`card-header`）

#### 文件结构
每个页面包含四个文件：
- `.js` - 页面逻辑
- `.json` - 页面配置
- `.wxml` - 页面模板
- `.wxss` - 页面样式

### 7.2 样式规范

#### 颜色定义
```css
/* 主色调 */
--primary-color: #d4a853;      /* 活力橙 */
--background-color: #0a0a1a;   /* 深炭黑 */
--card-background: #141430;    /* 卡片背景 */
```

## 9. 常见问题

### 9.1 开发问题

**Q1：为什么 tabBar 只能配置 5 个页面？**

A：微信小程序官方限制，tabBar 最多只能配置 5 个页面。风水分析页面通过总览页面的入口跳转访问。

**Q2：WXML 模板中为什么不能使用 JavaScript 方法？**

A：微信小程序的 WXML 模板语法有限制，不支持直接调用 JavaScript 方法（如 `.toFixed()`）。需要在 JS 中预处理数据，然后在模板中使用预处理后的字段。

**Q3：为什么使用 `wx.getDeviceInfo()` 替代 `wx.getSystemInfoSync()`？**

A：微信小程序基础库 3.15.0 起，`wx.getSystemInfoSync()` 已弃用，推荐使用新的 API：
- `wx.getDeviceInfo()` - 获取设备信息
- `wx.getAppBaseInfo()` - 获取应用基础信息
- `wx.getWindowInfo()` - 获取窗口信息
- `wx.getSystemSetting()` - 获取系统设置

### 9.2 功能问题

**Q1：命盘数据存储在哪里？**

A：命盘数据存储在微信小程序的本地存储中（`wx.setStorageSync`），数据仅在当前设备上保存。

**Q2：如何更新后端 API 地址？**

A：修改 `miniprogram/app.js` 中的 `globalData.apiBaseUrl` 字段。

**Q3：风水分析需要什么权限？**

A：风水分析需要获取用户位置信息，需要用户授权 `scope.userLocation` 权限。

### 9.3 部署问题

**Q1：如何配置云开发环境？**

A：在 `miniprogram/app.js` 中修改 `wx.cloud.init` 的 `env` 参数为你的云开发环境 ID。

**Q2：如何解决跨域问题？**

A：在后端 FastAPI 中配置 CORS：
```python
from fastapi.middleware.cors import CORSMiddleware

app.add_middleware(
    CORSMiddleware,
    allow_origins=["*"],
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)
```

**Q3：如何查看小程序日志？**

A：在微信开发者工具中，点击"调试器"面板的"Console"标签查看日志。
